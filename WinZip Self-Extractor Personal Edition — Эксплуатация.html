Чтобы создать самораспаковывающийся Zip-файл, вы должны начать с обычного
Zip-файла (расширение .zip), который вы создали с помощью WinZip®, скачали и т. д.
Затем вы можете либо
откройте файл с помощью WinZip и выберите «Создать файл .exe » в меню « Действия
» WinZip или
Щелкните правой кнопкой мыши файл Zip в «Моем компьютере» или в проводнике
Windows и выберите « Создать самораспаковывающийся файл (.Exe)» в контекстном
меню проводника WinZip.
Любое из этих действий активирует WinZip Self-Extractor Personal Edition, если вы не
указали другую программу в поле Make Exe на вкладке Program Locations диалогового
окна Configuration. Если вы решили создать значок WinZip Self-Extractor Personal
Edition при установке WinZip, вы также можете запустить программу, дважды щелкнув
этот значок.
Диалоговое окно WinZip Self-Extractor Personal Edition содержит следующие поля:
Создайте самораспаковывающийся Zip-файл из
Имя стандартного (не самораспаковывающегося) Zip-файла, который нужно
преобразовать в самораспаковывающийся Zip-файл.
Папка по умолчанию «Распаковать в»
Целевая папка по умолчанию для операции распаковки. Если оставить пустым,
используется папка, указанная пользовательской переменной окружения TEMP=.
Охватывающая поддержка
Съемные диски, такие как дискеты, имеют очень ограниченную емкость. Иногда
создаваемые вами самораспаковывающиеся Zip-файлы не помещаются на одну
дискету. В этом случае вы можете попросить WinZip Self-Extractor сохранить один файл
Zip на нескольких дискетах. Этот процесс создания Zip-файла, занимающего несколько
дисков, называется «растягиванием».
Когда вы распределяете Zip-файлы по нескольким дискам с помощью WinZip
Self-Extractor, во время извлечения не требуется никакого специального программного
обеспечения. Вашим пользователям просто предлагается вставить любые
дополнительные диски в набор из нескольких томов в соответствующее время во
время извлечения.
Следующие варианты определяют, хотите ли вы использовать функцию объединения
для создания одного самоизвлекателя более чем на одном съемном диске, и какой
метод использовать.
No Spanning — WinZip не будет создавать .EXE-файл, который охватывает съемные
диски.
Безопасный метод объединения — при выборе безопасного метода WinZip
Self-Extractor помещает на первый диск два файла: Zip-файл с первой частью сжатых
данных и небольшой файл .EXE с исполняемым кодом, который требуется для
извлечения данные. Второй и последующие диски содержат Zip-файлы с остальными
частями ваших сжатых данных. Безопасный метод используется по умолчанию и
является рекомендуемым методом.
Старый метод объединения. В отличие от этого, старый метод объединения помещает
на первый диск файл .EXE, который содержит как исполняемый код, используемый
для извлечения сжатых данных, так и первую часть этих данных. Второй и
последующие диски содержат оставшиеся части ваших сжатых данных в файлах,
которые (даже если они фактически не содержат никакого исполняемого кода) имеют
расширение EXE. Это потенциально опасно, потому что, если пользователь
непреднамеренно попытается запустить EXE-файл на втором или последующих
дисках, могут появиться случайные сообщения об ошибках и даже зависание системы.
Примечание: WinZip Self-Extractor не может использовать неформатированные диски.
Все диски, которые вы используете с WinZip Self Extractor, должны быть уже
отформатированы.
Перезаписать по умолчанию
Этот флажок определяет, должен ли по умолчанию быть установлен флажок
«Перезаписывать файлы без запроса» в диалоговом окне самораспаковывающегося
Zip-файла.
Нажмите OK , чтобы создать самораспаковывающийся Zip-файл. Обязательно
протестируйте самораспаковывающийся Zip-файл перед его распространением.
